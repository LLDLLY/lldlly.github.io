git message

  • feat: 新增feature
  • fix: 修复bug
  • docs: 仅仅修改了文档，如README.md
  • style: 仅仅是对格式进行修改，如逗号、缩进、空格等,不改变代码逻辑
  • refactor: 代码重构，没有新增功能或修复bug
  • perf: 优化相关，如提升性能、用户体验等
  • test: 测试用例，包括单元测试、集成测试
  • chore: 改变构建流程、或者增加依赖库、工具等
  • revert: 版本回滚
